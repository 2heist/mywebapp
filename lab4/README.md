# Infrastructure as Code (IaC) - Lab 4

Цей репозиторій містить автоматизоване розгортання трирівневої архітектури (Reverse Proxy -> Node.js App -> PostgreSQL DB) за допомогою Terraform та Ansible на базі гіпервізора KVM/QEMU.

## Структура проєкту
* `terraform/` — конфігураційні файли для створення віртуальних машин та віртуальної мережі.
* `ansible/` — плейбуки та ролі для налаштування ОС, баз даних, застосунку та Nginx.

## Вимоги до середовища
* Гіпервізор `libvirt` (QEMU/KVM).
* Встановлений `terraform`.
* Встановлений `ansible`.

## 1. Розгортання інфраструктури (Terraform)
Перейдіть у директорію `terraform` та ініціалізуйте провайдера:
```bash
cd terraform
terraform init
```

Запустіть створення ресурсів (2 віртуальні машини: `worker` та `db`):
```bash
terraform apply
```
*Після успішного виконання Terraform виведе IP-адреси створених машин.*

## 2. Налаштування конфігурації (Ansible)
Перед запуском перевірте, що IP-адреси в `ansible/inventory.ini` відповідають тим, що видав Terraform.

Перейдіть у директорію `ansible` та перевірте доступність серверів:
```bash
cd ../ansible
ansible all -m ping
```

Запустіть плейбук для конфігурації всього середовища:
```bash
ansible-playbook playbook.yml
```

## Функціонал
* **База даних:** PostgreSQL налаштовано на сервері `db`. Прямий доступ ззовні заблоковано (`pg_hba.conf` дозволяє підключення тільки з IP-адреси `worker`).
* **Веб-сервер:** Nginx на сервері `worker` працює як reverse-proxy для Node.js застосунку.
* **Health Checks:** Реалізовано ендпоінти `/health/alive` та `/health/ready` (перевіряє підключення до БД).
* **Користувачі:** Створено користувачів `teacher` та `operator`. Для `operator` налаштовано обмежені права `sudo` для керування сервісами `app` та `nginx`.
