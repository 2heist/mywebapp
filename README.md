# Лабораторна робота №1: Розгортання Web-сервісу з автоматизацією

## 1. Варіант індивідуального завдання
Розрахунки на основі номера заліковки (N = 6):
* **V2 = (6 % 2) + 1 = 1**: MariaDB, конфігурація через аргументи командного рядка.
* **V3 = (6 % 3) + 1 = 1**: Notes Service (простий сервіс для зберігання текстових нотаток).
* **V5 = (6 % 5) + 1 = 2**: Порт застосунку 5200.

## 2. Документація по розробленому веб-застосунку
* **Призначення:** Зберігання та перегляд текстових нотаток.
* **Налаштування та запуск:** Застосунок конфігурується через передачу аргументів (`--port`, `--dbuser`, `--dbpassword`, `--dbname`). Запускається автоматично через Systemd Socket Activation (порт 5200). 

### API Ендпоінти
* `GET /health/alive` — перевірка стану (завжди повертає 200 OK).
* `GET /health/ready` — перевірка підключення до бази даних.
* `GET /` — повертає HTML зі списком усіх ендпоінтів.
* `GET /notes` — список усіх нотаток (повертає JSON або HTML таблицю залежно від заголовку `Accept`).
* `POST /notes` — створення нової нотатки (приймає `title`, `content`).
* `GET /notes/<id>` — виведення повного вмісту конкретної нотатки.

## 3. Документація по розгортанню
* **Базовий образ:** Ubuntu Server 24.04 LTS (офіційний образ).
* **Вимоги до ресурсів:** 1 CPU, 1GB RAM, 10GB Disk.
* **Доступ до ВМ:** Вхід через консоль віртуальної машини (або SSH) користувачем `student` (або `teacher` з паролем `12345678`).
* **Запуск автоматизації:**
  1. Клонувати репозиторій.
  2. Виконати `chmod +x setup.sh`
  3. Запустити скрипт: `sudo ./setup.sh`

## 4. Інструкція з тестування
1. Перевірка Nginx: `curl http://localhost/`
2. Перевірка створення нотатки: `curl -X POST -H "Content-Type: application/json" -d '{"title":"Test", "content":"Hello"}' http://localhost/notes`
3. Перевірка статусу Systemd сервісу: `sudo systemctl status mywebapp.service`
