# Розгортання Web-сервісу та Контейнеризація (Лабораторні роботи №1 та №2)

## 1. Варіант індивідуального завдання
Розрахунки на основі номера заліковки (N = 6):
* **V2 = (6 % 2) + 1 = 1**: MariaDB, конфігурація через аргументи командного рядка.
* **V3 = (6 % 3) + 1 = 1**: Notes Service (простий сервіс для зберігання текстових нотаток).
* **V5 = (6 % 5) + 1 = 2**: Порт застосунку 5200.

## 2. Документація по розробленому веб-застосунку
* **Призначення:** Зберігання та перегляд текстових нотаток.
* **Налаштування та запуск:** Застосунок конфігурується через передачу аргументів (`--port`, `--dbuser`, `--dbpassword`, `--dbname`).

### API Ендпоінти
* `GET /health/alive` — перевірка стану (завжди повертає 200 OK).
* `GET /health/ready` — перевірка підключення до бази даних.
* `GET /` — повертає HTML зі списком усіх ендпоінтів.
* `GET /notes` — список усіх нотаток (повертає JSON або HTML таблицю залежно від заголовку `Accept`).
* `POST /notes` — створення нової нотатки (приймає `title`, `content`).
* `GET /notes/<id>` — виведення повного вмісту конкретної нотатки.

## 3. Розгортання за допомогою Docker Compose (Лабораторна робота №2)

Цей проєкт повністю контейнеризовано. Автоматичне розгортання інфраструктури реалізовано за допомогою Docker Compose і складається з трьох ізольованих сервісів: Node.js застосунку, бази даних MariaDB та веб-сервера Nginx (Reverse Proxy).

### Вимоги
* Docker
* Docker Compose

### Інструкція з запуску
1. Переконайтеся, що порт 80 на хост-машині не зайнятий іншими веб-серверами.
2. Виконайте команду в кореневій папці проєкту для збирання образів та запуску у фоновому режимі:
   ```bash
   docker compose up --build -d
   ```
3. Після успішного запуску застосунок буде доступний у браузері за адресою: `http://localhost`

### Керування контейнерами
* Перегляд логів усіх сервісів: `docker compose logs -f`
* Зупинка сервісів із збереженням даних БД на диску (volumes): `docker compose down`

## 4. Локальне розгортання через скрипт (Лабораторна робота №1 - Legacy)

* **Базовий образ:** Ubuntu Server 24.04 LTS.
* **Вимоги до ресурсів:** 1 CPU, 1GB RAM, 10GB Disk.
* **Доступ до ВМ:** Вхід через консоль віртуальної машини (або SSH) користувачем `student` (або `teacher` з паролем `12345678`).
* **Запуск автоматизації:**
  1. Клонувати репозиторій.
  2. Виконати `chmod +x setup.sh`
  3. Запустити скрипт: `sudo ./setup.sh`
  4. Застосунок запускатиметься автоматично через Systemd Socket Activation.

## 5. Інструкція з тестування

Незалежно від обраного методу розгортання, працездатність сервісу можна перевірити наступними командами:

1. **Перевірка Nginx та головної сторінки:**
   ```bash
   curl http://localhost/
   ```
2. **Перевірка створення нотатки:**
   ```bash
   curl -X POST -H "Content-Type: application/json" -d '{"title":"Test", "content":"Hello"}' http://localhost/notes
   ```
3. **Перевірка статусу Systemd сервісу (тільки для розгортання за ЛР №1):**
   ```bash
   sudo systemctl status mywebapp.service
   ```
